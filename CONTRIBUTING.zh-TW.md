# 參與貢獻 Open Live Writer Evolution

Open Live Writer Evolution 是一個開源專案，竭誠歡迎社群成員的參與和貢獻。

本專案已採用 [貢獻者公約 (Contributor Covenant)](http://contributor-covenant.org/) 定義的行為準則，以明確規範社群中的預期行為。

## 回報問題 (Issues)

如果您發現了 Bug，或者對新功能/改進有任何想法，請先查看 [Issues](https://github.com/quicktop/OpenLiveWriterEvolution/issues)。您可以使用篩選框來搜尋是否有人提出過類似的問題或想法。如果您遇到的問題是全新的，請建立一個新的 Issue。

建立 Issue 時，請提供儘可能詳細的資訊。建議提供以下內容：

- 您正在搭配 Live Writer 使用的部落格伺服器類型（例如：Blogger、WordPress 等），或是您嘗試發布文章的部落格網址。
- 您正在執行的作業系統版本（例如：Windows 10、Windows 11）。
- 能重現您所發現錯誤的詳細步驟。
- 您看到的任何錯誤詳細資訊；錯誤對話框的螢幕截圖非常有幫助。您可以點選選單中的「檔案」→「關於 Open Live Writer」→「顯示記錄檔」，來取得您的 Open Live Writer Evolution 記錄（Log）檔案。如果您能看到與您問題相關的錯誤訊息，請一併提供其詳細內容。

## 說明文件

Open Live Writer Evolution 主要有三種類型的說明文件，我們非常期待您能在此提供協助：

1. **一般使用者文件**：介紹如何充分利用 Open Live Writer Evolution。
2. **外掛開發人員文件**：關於如何為 Open Live Writer Evolution 撰寫外掛的詳細說明與範例。
3. **開發人員文件**：介紹如何建置、測試以及提交新的 Bug 修正或功能。

如果您想為說明文件做出貢獻，請按照以下指示發送 Pull Request (PR) 來提交您的修改。

## 開發人員

我們非常歡迎您共同參與，讓 Open Live Writer Evolution 變得更好。若要深入瞭解開發細節，請參閱 [CLAUDE.md](CLAUDE.md) 檔案以取得建置說明和架構概觀。

為本專案貢獻程式碼非常簡單：

1. 派生（Fork）此儲存庫並複製（Clone）到本機。
2. 建立一個特定的主題分支（Topic Branch），新增您實作的功能或修正 Bug。
3. 發送 Pull Request (PR) 共享您的成果！

如果您想在開始動工之前，先討論您計畫進行的變更，歡迎隨時提出 Issue，或在 [GitHub Discussions](https://github.com/quicktop/OpenLiveWriterEvolution/discussions) 發起討論。
